## 2. Типографика (Fluid)

Для обеспечения адаптивности типографики будет использоваться `clamp()`.

```css
/* Пример Fluid Typography */
h1 {
  font-size: clamp(2rem, 5vw, 3rem);
}
```

## 3. Сетка и Container Queries

*   **Сетка:** 12-колоночная сетка на базе CSS Grid.
*   **Container Queries:** Использовать для компонентов, которые должны адаптироваться к размеру своего контейнера, а не всего экрана.

```css
/* Пример Container Queries */
.card-container {
  container-type: inline-size;
}

@container (min-width: 400px) {
  .card {
    flex-direction: row;
  }
}
```

## 4. Состояния компонентов

Все интерактивные компоненты должны иметь четко определенные состояния:

| Состояние | Описание | Пример CSS |
| :--- | :--- | :--- |
| **Hover** | Наведение курсора | `filter: brightness(1.1);` |
| **Focus** | Фокус с клавиатуры | `outline: 2px solid var(--colors-primary-base);` |
| **Disabled** | Неактивное состояние | `opacity: 0.5; cursor: not-allowed;` |
| **Error** | Ошибка валидации | `border: 1px solid red;` |
| **Empty** | Пустое состояние (нет данных) | Отображение специального плейсхолдера |
| **Skeleton** | Загрузка данных | Использование skeleton-компонентов |

## 5. Чек-лист доступности (a11y - WCAG 2.2 AA)

1.  **Контраст:** Весь текст должен иметь контраст не менее `4.5:1`.
2.  **Навигация с клавиатуры:** Все интерактивные элементы доступны через `Tab`.
3.  **Видимый фокус:** `focus-ring` должен быть всегда виден.
4.  **Семантический HTML:** Использовать корректные теги (`<nav>`, `<main>`, `<button>`).
5.  **ARIA-атрибуты:** Использовать `aria-live` для асинхронных обновлений и `aria-label` для иконок.
6.  **Альтернативный текст:** Все изображения должны иметь `alt` атрибут.
7.  **Масштабирование:** Сайт должен быть полностью функционален при увеличении до 200%.
