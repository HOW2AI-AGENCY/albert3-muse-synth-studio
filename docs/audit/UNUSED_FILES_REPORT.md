# Отчет о неиспользуемых файлах и "спящем" функционале

**Дата:** 27 ноября 2025 г.

## 1. Обзор

В ходе статического анализа кодовой базы были выявлены файлы, которые не импортируются и не используются в приложении, а также файлы, содержащие полнофункциональную, но отключенную логику ("спящий" функционал).

## 2. Кандидаты на удаление (мертвый код)

Эти файлы не имеют ссылок в проекте и, скорее всего, являются остатками старой логики. Рекомендуется удалить их после дополнительной быстрой проверки.

| Файл | Описание | Приоритет |
| :--- | :--- | :--- |
| **`src/utils/offline.js`** | Содержит пустую функцию-заглушку `offlineHandler`. Не используется нигде в проекте. | **Высокий** |

### План действий

1.  Создать задачу "Удалить неиспользуемый файл `src/utils/offline.js`".
2.  Назначить ее автору модуля (если возможно определить) или тимлиду для финального подтверждения.
3.  Удалить файл в рамках спринта по очистке техдолга.

## 3. "Спящий" функционал (требует ревью продукта)

Эти файлы содержат работающий, но отключенный на уровне конфигурации или кода функционал. Перед удалением или активацией требуется решение от владельца продукта (Product Owner).

| Файл | Описание | Как отключен | Рекомендация |
| :--- | :--- | :--- | :--- |
| **`src/utils/serviceWorker.ts`** | Содержит полнофункциональный менеджер Service Worker для PWA, включая кеширование аудио, оффлайн-доступ и механизм обновлений. | Закомментирован вызов `initServiceWorker()` в `src/main.tsx`. | **Провести ревью с PO.** Необходимо решить: нужен ли приложению оффлайн-функционал? Если да — создать эпик на его полноценное тестирование и включение. Если нет — удалить код, чтобы не вводить разработчиков в заблуждение. |

### План действий

1.  Создать задачу "Провести ревью "спящего" функционала Service Worker" для Product Owner.
2.  Приложить к задаче данный отчет и описание возможностей, которые предоставляет `serviceWorker.ts`.
3.  На основе решения PO либо запланировать работы по активации, либо удалить связанный код.

## 4. Дальнейшие шаги

- Провести аналогичный анализ для других директорий, в особенности `src/components`, `src/hooks` и `src/services`.
- Внедрить в CI/CD пайплайн инструмент типа `depcheck` или `unimported` для автоматического обнаружения неиспользуемых файлов и зависимостей на постоянной основе.
