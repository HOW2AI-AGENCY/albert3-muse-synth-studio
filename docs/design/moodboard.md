# Moodboard: Native Mobile UI Modernization

**Цель:** Создать профессиональный, современный и интуитивно понятный дизайн интерфейса с акцентом на нативный мобильный опыт (iOS/Android).

---

## 1. Визуальный стиль и эстетика

*   **Чистота и минимализм:**
    *   Акцент на контенте, минимум отвлекающих элементов.
    *   Простые, четкие линии и формы.
    *   Достаточное количество "белого пространства" для улучшения читаемости и фокуса.
*   **Современная типографика:**
    *   Использование системных шрифтов (San Francisco для iOS, Roboto для Android) или близких к ним, с хорошей читаемостью на малых экранах.
    *   Четкая иерархия заголовков и текста.
*   **Глубина и слои (iOS-like):**
    *   Использование легких теней и размытия (backdrop-filter) для создания ощущения слоев и иерархии, особенно для модальных окон, BottomSheet и навигационных панелей.
    *   Эффект "стекла" (glassmorphism) для некоторых элементов, чтобы придать современный вид без излишней сложности.
*   **Яркие акценты:**
    *   Ограниченное использование ярких, но не кричащих цветов для интерактивных элементов (кнопки, иконки, индикаторы прогресса).
    *   Градиенты для ключевых элементов (например, заголовков, кнопок CTA) для придания динамичности.

---

## 2. Цветовая палитра

*   **Основные цвета:**
    *   Нейтральные, приглушенные фоновые цвета (темно-серый, почти черный для темной темы; светлый серый, белый для светлой темы).
    *   Контрастные цвета для текста и иконок для обеспечения читаемости.
*   **Акцентные цвета:**
    *   Один-два ярких, но приятных глазу цвета (например, глубокий синий, изумрудный, фиолетовый) для интерактивных элементов и индикаторов.
    *   Использование градиентов с этими акцентными цветами.
*   **Семантические цвета:**
    *   Четкие цвета для статусов (успех, ошибка, предупреждение).

---

## 3. Иконография

*   **Единообразный стиль:**
    *   Использование одной библиотеки иконок (например, Hugeicons, Lucide React) с консистентным стилем (stroke, solid, duotone).
    *   Предпочтение иконок с закругленными углами для более "дружелюбного" вида.
*   **Понятность:**
    *   Иконки должны быть интуитивно понятны и не требовать дополнительных пояснений.
    *   Достаточный размер иконок для удобного нажатия на мобильных устройствах (touch targets).

---

## 4. Компоненты и паттерны (Native-inspired)

*   **Навигация:**
    *   **Bottom Tab Bar (мобильная):** Чистая, минималистичная, с активными индикаторами и тактильной обратной связью. Увеличенные touch targets.
    *   **Top App Bar (мобильная):** Контекстная, с заголовком и кнопками действий.
    *   **Sidebar (десктоп):** Компактный, с возможностью расширения, с иконками и подсказками.
*   **Карточки:**
    *   Легкие тени, закругленные углы, четкая иерархия контента.
    *   Интерактивные состояния (hover, active) с плавными переходами.
*   **Кнопки:**
    *   Различные варианты (primary, secondary, ghost, icon) с четкими состояниями (disabled, loading, hover, active).
    *   Увеличенные touch targets для мобильных.
*   **Поля ввода:**
    *   Чистый, минималистичный дизайн, четкие лейблы и плейсхолдеры.
    *   Визуальная обратная связь при фокусе и ошибках.
*   **Диалоги/Модальные окна:**
    *   **BottomSheet (мобильная):** Для детальных панелей и контекстных действий.
    *   **Responsive Dialog:** Адаптивный дизайн для всех разрешений.
    *   Эффект размытия фона (backdrop-filter) для создания фокуса.
*   **Индикаторы:**
    *   Прогресс-бары, спиннеры, скелетоны для обозначения загрузки и длительных операций.
    *   Тактильная обратная связь (haptic feedback) для интеракций.

---

## 5. Анимации и микроинтеракции

*   **Плавные переходы:**
    *   Использование `framer-motion` или CSS-переходов для плавных изменений состояния (например, при наведении на карточку, переключении вкладок).
*   **Микроинтеракции:**
    *   Легкие анимации при нажатии на кнопки, переключении тумблеров, появлении уведомлений.
    *   Визуальная обратная связь при успешных/неудачных действиях.
*   **Оптимизация:**
    *   Использование GPU-ускоренных свойств (transform, opacity) для плавности.

---

## 6. Доступность (Accessibility)

*   **Контраст:** Соответствие WCAG AAA для текста и UI-элементов.
*   **Touch Targets:** Все интерактивные элементы ≥ 44x44px.
*   **ARIA-метки:** Использование ARIA-атрибутов для улучшения доступности для скринридеров.
*   **Фокус:** Четкие индикаторы фокуса для навигации с клавиатуры.

---

## 7. Мобильная специфика

*   **Safe Area:** Учет безопасных зон (notch, home indicator) на iOS.
*   **Жесты:** Интеграция свайпов для навигации или действий (например, свайп для закрытия BottomSheet).
*   **Производительность:** Оптимизация скорости загрузки и отзывчивости для мобильных сетей и устройств.
*   **Ландшафтный режим:** Проверка корректного отображения интерфейса в горизонтальной ориентации.
